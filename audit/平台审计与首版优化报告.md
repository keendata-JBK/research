# 科杰战略情报与认知共创平台审计与首版优化报告

日期：2026-07-03

## 1. 审计范围

本次覆盖首页、竞对分析、行业分析、观点提炼、对科杰启发、公众号材料导入、领导 3 分钟简报和移动端导航。依据包括：

- `CODEX_OPERATION_GUIDE.md`
- 《科杰战略情报与认知共创平台产品设计方案》
- `Map1.pdf` 中“数据重构：AI 产业化时代的新基建”八条观点主线
- 科杰科技官方 Logo
- 本轮浏览器现状截图与优化后截图

## 2. 现状结论

原版本已具备 React 页面、雷达 JSON、领导简报和 GitHub Actions 骨架，但核心体验仍是“展示静态卡片”，没有形成“材料进入 → 事实证据 → 行业共性 → 观点推导 → 科杰启发 → 产品动作/母版候选”的可操作闭环。

### 关键问题

1. 竞对对象写死，不能在页面新增、删除或绑定公众号/官网来源；对象数量也不足以覆盖模型、算力、云、数据基础设施和产业链主。
2. 行业分析只有平铺主题，没有主题选择、热度变化、厂商表达对照、证据来源或转观点动作。
3. 观点提炼只有“外部表达/科杰表达”结果，缺少来源、推导链、置信度、使用场景和 Map1 主线。
4. 对科杰启发高度同质化，没有责任人、优先级、当前差距、下一版本动作，无法进入产品规划。
5. 公众号链接输入框存在，但必须粘贴正文才能提交，实际不能“只输入链接”。
6. 使用字母 K 代替公司 Logo，与官方品牌资产不一致。
7. 移动端菜单可用，但部分桌面结构只是纵向堆叠，缺少针对小屏的动作布局与信息优先级。

## 3. 首版优化内容

### 3.1 竞对分析

- 种子对象扩展到 25 家，覆盖模型、云、算力/芯片、数据基础设施、专业软件和数据应用。
- 增加“添加竞合对象”表单，支持名称、类型、状态、公众号/官网来源。
- 自定义对象保存在当前浏览器，可删除；共享自动扫描仍需同步仓库 `watchlist.json`。
- 公司详情增加信息源、事件时间线，以及“来源事实 → 初步影响 → 建议动作”下钻。
- 增加“导入该对象公众号”入口，自动带入关联公司。

### 3.2 行业分析

- 改为主题工作台：左侧主题趋势，右侧主题详情。
- 展示热度、变化、共同表达、代表厂商、科杰角度、厂商表达对照和关联证据。
- 增加“转为观点候选”动作，将主题沉淀到观点池。

### 3.3 观点提炼与 Map1

- 增加观点列表和详情下钻。
- 展示外部原始表达、推导节点、科杰化表达、适用场景、来源证据和置信度。
- 把 Map1 重构为八条可交互观点地图：时代判断、核心矛盾、数据重构、新基建、三层重构、算数模行、落地方法、终章表达。
- 支持复制科杰化表达及加入母版候选。

### 3.4 对科杰启发与产品升级

- 增加战略启发/产品升级双视图。
- 产品路线按“外部迭代信号 → 当前差距 → 下一版本动作 → 责任人/优先级”组织。
- 首版覆盖 KeenData Lakehouse、企业认知模型、KeenClaw、可信数据空间和国产生态适配。

### 3.5 公众号导入

- 增加“公众号/网页链接”和“复制正文”两种模式。
- 链接模式允许只输入 URL；系统先登记来源和关联对象，状态为“待补全文”。
- 对无正文链接不生成高置信正式判断，明确提示 GitHub Pages 跨域和公众号反爬限制。
- 正文模式继续生成竞对事件、行业主题、观点和科杰启发候选。

### 3.6 首页、简报与品牌

- 首页突出 3 分钟简报、真实统计、产品升级信号和领导讨论议题。
- 主导航正式加入“3分钟简报”，并保留独立 `leader.html`。
- 主应用和独立简报均替换为官方科杰科技 Logo。
- 增加“AI 生成、人工审核后发布”的全局候选分析提示。

## 5. 仍需明确的生产边界

1. GitHub Pages 是静态站点，自定义对象和前端导入结果当前保存在浏览器 `localStorage`，不是团队共享数据库。
2. 微信公众号全文无法在纯浏览器环境稳定抓取。当前正确策略是链接登记、复制正文或 GitHub Actions/Issue Form 异步处理，不能绕过反爬。
3. 公开 GitHub Pages 不适合承载敏感战略材料；正式内部使用应增加访问控制并迁移到带认证和数据库的部署形态。
4. 截图审计只能确认可见布局与基本交互，键盘全流程、读屏语义和完整 WCAG 合规仍需专项测试。

## 6. 下一阶段建议

P1：把页面新增对象和公众号链接提交到 GitHub Issue Form/Workflow，形成团队共享 watchlist 和材料队列；补充人工审核状态流转和导出。

P2：接入受控 LLM 分析服务，强制输出来源、证据片段、事实/判断/建议类型和置信度；再迁移到有权限体系的正式生产部署。
