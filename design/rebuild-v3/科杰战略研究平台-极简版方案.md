# 科杰战略研究平台｜极简版方案

## 1. 产品原则

平台不是研究后台，而是领导随时打开即可阅读的动态战略简报。前台只呈现三个业务标签：

1. 竞合分析
2. 行业发展趋势
3. 对科杰的启发

全局只保留两个主要动作：`导入公众号`、`导出最新 PDF`。采集任务、证据核验、主题归并、观点提炼、版本管理等复杂流程由后台自动完成，不占用领导阅读界面。

## 2. 最小使用闭环

```text
粘贴公众号链接
  → 自动识别公众号、文章、时间与涉及主体
  → 自动生成竞合动态并归入主体时间线
  → 自动更新行业趋势
  → 自动更新对科杰的启发
  → 一键导出最新战略分析 PDF
```

领导的操作不超过四种：切换标签、选择主体、点开摘要、导出 PDF。

## 3. 页面结构

### 顶部固定栏

- 左侧：科杰科技 Logo + `科杰战略研究`
- 中间：3 个标签
- 右侧：搜索、`导入公众号`、`导出最新 PDF`

不设置左侧多级导航，不展示系统模块、任务队列和复杂统计。

### 标签一：竞合分析

默认首页。内容由“主体选择 + 时间流”组成。

- 主体支持搜索、新增和分组：国际厂商、国内数据厂商、模型厂商、算力厂商、合作伙伴。
- 每篇公众号文章自动识别一个或多个主体，并形成带时间戳的动态。
- 每条动态仅展示：日期、主体、事件标题、一句话判断、影响等级、来源。
- 点击后展开：事实摘要、关键表达、战略意图、对科杰影响、原文链接。
- 相同事件多篇报道自动合并，保留全部来源。
- 主体不存在时自动创建候选，用户确认后加入竞合对象库。

### 标签二：行业发展趋势

不展示文章堆积，只展示已经形成的趋势判断。

- 顶部显示 `本周新增 / 持续增强 / 出现分歧` 三种变化状态。
- 每条趋势包含：一句话判断、近 30 天变化、代表厂商、证据数量、更新时间。
- 点击展开代表表达、主要分歧、时间演化和来源。
- 支持按时间范围与行业主题筛选，但默认保持全局视图。

### 标签三：对科杰的启发

只呈现可讨论、可决策的公司启发，固定为六个主题：

1. 核心定位
2. 企业认知模型
3. 产品与方案架构
4. 可信数据空间
5. 产品升级路线
6. 生态与市场打法

每条启发包含：建议表达或动作、为什么、来自哪些竞合/趋势、更新时间、是否已进入 PDF。

## 4. 公众号导入

点击 `导入公众号` 只出现一个轻量弹窗：

1. 粘贴 `mp.weixin.qq.com` 链接。
2. 点击 `开始解读`。
3. 系统显示识别结果：文章标题、公众号、发布时间、涉及主体、建议归类。
4. 用户点击 `确认加入`，或直接关闭让系统按默认规则保存。

后台自动完成：正文抓取、失败回退、去重、主体识别、事件拆分、摘要、观点与启发更新。抓取失败时只要求用户粘贴正文或上传截图，不暴露技术错误。

## 5. 最新 PDF 导出

点击 `导出最新 PDF` 后直接生成，不要求用户配置复杂选项。默认报告名称：

`科杰AI数据基础设施战略与竞合格局分析报告_YYYYMMDD.pdf`

报告沿用现有 V6.1 的 A4 版式、事实/判断/建议分层、图表与参考文献体系，并吸收 V3 的领导速读结构：

1. 封面与研究截止日期
2. 董事长一分钟速读
3. 竞合格局与关键主体动态
4. 行业发展趋势
5. 对科杰的启发与决策议题
6. 产品升级与阶段行动建议
7. 研究边界、参考文献与证据清单

导出弹窗只展示：数据截止时间、最新纳入内容数、报告页数预估、生成按钮。生成完成后保留版本号和导出记录。

## 6. 前后台边界

### 领导前台可见

- 3 个标签
- 公众号导入
- 原文下钻
- 搜索
- 最新 PDF 导出

### 自动后台处理

- 抓取、OCR、去重和来源快照
- 主体识别与主体候选创建
- 事实/推断/建议分层
- 竞合事件、趋势与启发的级联更新
- 证据质量、冲突和时效检查
- PDF 章节映射、引用编号与版本生成

必要的人工审核通过小红点、待确认提示或导入结果完成，不创建单独“审核中心”标签。

## 7. 推荐视觉方向

推荐采用“极简双栏”：顶部 3 标签，左侧轻量主体列表，右侧连续时间流；点击动态后在右侧抽屉展开。它对领导最容易理解，也能容纳未来主体增长。行业趋势与科杰启发沿用相同的列表—详情交互，避免每个标签采用不同操作方式。
