# 科杰战略研究平台 V2.0 重构方案

> 产品定位：让外部信息持续进入、被证据化分析，并自动转化为科杰可复用的战略观点、产品升级建议与动态研究报告。
>
> 本轮范围：产品方案与视觉方向，不进入正式开发。

## 1. 重构结论

当前平台的核心问题不是页面数量不够，而是缺少一条可持续运转的研究生产链。多个模块彼此并列，用户看到的是结果卡片，却不知道材料从哪里来、为什么得到这个结论、如何继续下钻、怎样影响科杰报告，也无法判断内容是否已审核。

V2.0 不再以“竞对分析、行业分析、观点提炼、科杰启发”四个静态栏目组织产品，而以一条研究闭环组织：

```text
信源进入 → 内容解析 → 证据核验 → 竞对事件 → 行业主题
        → 观点候选 → 科杰启发 → 人工审核 → 动态报告更新
```

每一条结论必须可回到原文，每一条观点必须能解释由哪些证据推导，每一条科杰启发必须能说明影响哪个定位、产品、方案或报告章节。

## 2. 产品北极星

平台的核心产品不是“情报卡片”，而是一本持续进化、随时可追溯的《科杰战略研究报告》。

- 领导打开首页，3 分钟内知道今天发生了什么、为什么重要、对科杰意味着什么。
- 研究人员导入一条公众号链接后，系统自动形成结构化竞对事件、行业表达、观点候选和科杰启发。
- 产品与售前人员可以从启发直接下钻到证据，并将结论加入产品升级路线、方案母版或报告章节。
- 报告不是被 AI 静默改写，而是展示“候选变更—证据—影响章节—审核人—发布时间”的完整过程。

## 3. 用户与权限

| 角色 | 主要任务 | 默认权限 |
|---|---|---|
| 领导 | 速读、追问、批注、确认重点议题 | 查看全部已发布内容，批示与置顶 |
| 研究编辑 | 配置信源、复核证据、编辑观点、发布报告 | 内容全流程管理 |
| 产品/售前专家 | 补充判断、认领启发、进入路线图或方案母版 | 编辑业务判断与行动建议 |
| 普通共创者 | 导入材料、收藏、评论、提交候选观点 | 新增与协作，不直接发布 |
| 系统管理员 | 成员、权限、抓取任务、模型与审计配置 | 系统级管理 |

## 4. 核心信息对象

V2.0 统一以八类对象串起所有页面：

1. **信源 Source**：公众号、官网、新闻、PDF、研报、内部材料、人工记录。
2. **证据 Evidence**：原文片段、发布日期、作者、链接、截图、来源等级和可信度。
3. **竞对事件 Competitor Event**：某家公司在某个时间发生的产品、合作、案例、组织或话术变化。
4. **行业主题 Industry Theme**：跨多个公司的共同表达、趋势、分歧与演化阶段。
5. **观点 Viewpoint**：由多条证据支持，可被反驳、可版本化的判断。
6. **科杰启发 Kejie Implication**：观点对科杰定位、企业认知模型、产品架构、可信数据空间、行业方案、生态与销售表达的影响。
7. **行动 Action**：产品升级、竞合跟进、材料更新、验证实验和负责人。
8. **报告章节 Report Chapter**：被批准的结论进入动态研究报告后的落点与版本差异。

对象间关系固定为：

```text
信源 1..N 证据 → 竞对事件/行业主题 → 观点 → 科杰启发 → 行动/报告章节
```

## 5. 信息架构

### 5.1 一级导航

1. **今日研究**：领导首页，展示新信号、关键判断、科杰启发、待决议题和报告变化。
2. **信源中心**：公众号/官网订阅、链接导入、文件导入、采集任务、失败队列。
3. **竞对时间线**：按公司、类型、日期、重要度和合作状态查看可下钻事件。
4. **行业主题**：跨友商聚合表达、共识、差异、趋势强度和证据覆盖度。
5. **观点工坊**：AI 候选、证据链、反证、专家批注、合并与版本升级。
6. **科杰启发**：按六类启发组织，并可进入产品路线、方案母版或报告。
7. **动态报告**：章节树、当前版本、候选变更、差异比较、发布与导出。
8. **审核中心**：统一处理待核验、待发布、冲突证据和低置信度内容。

搜索不是独立模块，而是覆盖全部对象的全局能力；设置、成员和审计日志放入个人菜单或管理区。

### 5.2 首页优先级

首页不再平铺统计数字，首屏只回答五个问题：

1. 今天最重要的三条新信号是什么？
2. 哪个行业判断正在发生变化？
3. 哪些变化会影响科杰？
4. 动态报告新增或修订了什么？
5. 哪三件事需要领导判断或团队跟进？

## 6. 四个核心业务模块

### 6.1 竞对分析：从文章收藏变成带时间戳的事件流

**输入方式**

- 粘贴微信公众号文章链接。
- 订阅公众号或维护公众号清单，由采集任务定时发现新文章。
- 粘贴官网、新闻或研报链接。
- 上传 PDF、图片、网页归档或复制正文。
- 当公众号正文抓取失败时，明确提示原因，并自动切换为“复制正文/上传截图”补充模式。

**自动产物**

- 公司、发布日期、事件类型、标题和事实摘要。
- 关键原话与可复用表达。
- 产品/方案变化、战略意图、市场影响。
- 对科杰的影响与建议动作。
- 证据片段、原文链接、来源等级、置信度和审核状态。

**关键交互**

- 时间线可按公司、主题、事件类型、重要度筛选。
- 点击事件进入抽屉或详情页，展示原文证据、历史同类事件和变化趋势。
- 支持“加入行业主题”“提升为观点”“生成科杰启发”“加入报告候选”。
- 每个外部链接均可点击原站打开；失效链接保留抓取快照与证据片段。

### 6.2 行业分析：从公司列表变成跨来源主题综合

行业页以主题而不是文章为中心。每个主题包含：

- 当前共识：多数厂商正在如何表达。
- 关键分歧：不同类型厂商的路线差异。
- 表达演化：过去 30/90/180 天的用词和产品叙事变化。
- 代表证据：按来源等级和时间排序的原文片段。
- 厂商矩阵：谁在讲、讲到什么深度、是否已有产品或客户证据。
- 科杰位置：科杰当前表达与行业表达的相同点、空白和可建立差异化的位置。

默认主题包括 AI 数据基础设施、Agent-ready Data Platform、企业认知模型、可信数据空间、高质量数据集、AI Factory、国产算力适配、语义治理、Agent 控制平面、产业智能平台等；研究编辑可以新增、合并或归档主题。

### 6.3 观点提炼：把 Map1 变成可计算、可下钻的论证结构

Map1 不再只作为一张图展示，而是作为观点模板：

```text
时代判断 → 核心矛盾 → 核心答案 → 新基建定位
       → 三层架构 → 科杰架构 → 落地方法 → 终章表达
```

每个观点页面展示：

- 一句话判断。
- 推导逻辑和所处 Map1 节点。
- 支持证据、反对证据、证据时间范围和来源覆盖。
- 从哪些竞对事件和行业主题提升而来。
- 相比上一版本新增、修订或被削弱了什么。
- 适用场景：领导汇报、战略定位、行业研究、产品规划、销售表达。
- “升格为核心观点”“保留观察”“退回补证”三个决策。

系统只能自动生成观点候选，不能未经审核直接替换核心观点。

### 6.4 对科杰的启发：从静态文案变成定位、产品和行动系统

启发页至少包含六个长期主题：

1. 进一步强化科杰的核心定位。
2. 对“企业认知模型”的表达升级。
3. 对科杰产品/方案架构的启发。
4. 对可信数据空间的启发。
5. 对产品升级与版本路线的启发。
6. 对生态竞合、行业方案和销售表达的启发。

每条启发必须显示：来源观点、影响对象、建议动作、负责人、优先级、验证状态和报告落点。对于产品升级，系统增加“竞对最近版本—新增能力—客户场景—科杰现状—建议优先级”对比视图，避免只有抽象表达。

## 7. 动态研究报告

动态报告沿用既有研究分析报告工作流，但将一次性产出拆成在线增量流程：

```text
研究简报 → 证据台账 → 命题/反命题 → 章节映射
       → 候选修改 → 人工复核 → 发布新版本 → PDF/Word 导出
```

报告中心必须提供：

- 章节目录与章节负责人。
- 当前已发布版和工作版。
- 本周新增证据、本周新增观点、本周修改章节。
- 逐段差异比较，显示修改原因和支撑证据。
- 版本冻结、回滚、发布说明和导出。
- 日报、周报、专题报告与主报告共用同一证据底座，避免重复生成互相冲突的结论。

## 8. 自动更新与质量控制

### 8.1 自动化节奏

- 公众号/官网订阅：每 2–4 小时发现一次新内容。
- 新材料解析：进入队列后自动去重、抽取、分类和生成候选。
- 行业主题：每日增量聚合，每周生成趋势变化摘要。
- 科杰启发：新高重要度观点发布后重算；每日汇总候选。
- 动态报告：每日生成候选差异，周度由研究编辑发布稳定版本。
- 领导简报：工作日定时生成，同时支持手动刷新。

### 8.2 质量门槛

- 事实、推断、建议三层内容必须明显区分。
- 重要结论优先要求两个独立来源；单一来源必须提示。
- 所有结论保留原文链接、证据片段、抓取时间和来源等级。
- 自动去重识别转载文章和同一事件的多篇报道。
- 冲突证据不自动覆盖，而是进入“待判断”。
- 过期信息显示时效标签；依赖旧证据的观点自动降级。
- AI 只能创建候选，核心观点、科杰启发和报告发布保留人工审核。

## 9. 技术方案

### 9.1 推荐架构

```text
React/Vite Web + 移动响应式
        ↓
Supabase Auth / Postgres / Storage / Realtime
        ↓
采集与解析服务（Edge Function 或独立 Worker）
        ↓
任务队列 + 定时调度 + AI 分析流水线
        ↓
证据台账 / 知识对象 / 报告版本库
```

现有 GitHub Pages 只适合托管前端，不能承担公众号抓取、定时任务、密钥保护和长时 AI 处理。正式版本应将前端继续部署在静态站点或 Vercel，将认证、数据库、存储放在 Supabase，将公众号/网页抓取和 AI 任务放在服务端 Worker。公众号有反爬与版权限制，因此采用“自动解析 + 失败后复制正文/截图补充 + 保留原文链接”的双通道策略。

### 9.2 数据表建议

- `sources`、`source_subscriptions`、`ingestion_jobs`
- `documents`、`evidence_fragments`、`source_snapshots`
- `organizations`、`competitor_events`
- `industry_themes`、`theme_evidence_links`
- `viewpoints`、`viewpoint_versions`、`viewpoint_evidence_links`
- `kejie_implications`、`actions`
- `report_documents`、`report_chapters`、`report_revisions`
- `reviews`、`comments`、`notifications`、`audit_logs`

所有业务表启用行级权限；外部发布页面与内部工作区分开；服务端密钥不进入浏览器。

## 10. 交互原则与视觉方向

- 先展示“判断”，再展示“证据”，最后展示“动作”，避免统计卡片抢占首屏。
- 列表默认是一张连续研究台账，不把每一行都做成独立卡片。
- 关键对象支持抽屉下钻，保留用户当前上下文。
- 所有来源、公司、主题、观点、启发和报告章节都可点击跳转。
- 核心状态统一：`待解析 / 待核验 / 候选 / 已发布 / 已归档 / 有冲突`。
- 品牌使用科杰深蓝、亮蓝和大面积白色；减少渐变、阴影和装饰性大卡片。
- Map1 以论证路径和节点关系进入产品，不原样铺成静态大图。
- 领导端以简报和决策为主，研究端以证据与编辑为主，但共用同一数据。

## 11. 分期实施

### P0：研究闭环可用

- Supabase 登录、组织与权限。
- 公众号链接/正文/PDF 导入和失败回退。
- 竞对事件时间线与详情证据。
- 行业主题聚合。
- Map1 观点模板和候选审核。
- 六类科杰启发。
- 动态报告章节映射与版本差异。
- 首页领导简报和全局链接跳转。

### P1：自动订阅与协作

- 公众号/官网定时发现、任务队列、通知。
- 评论、批注、认领、审核流。
- 产品升级对比、行动跟踪。
- 日报/周报和移动端领导视图。

### P2：知识增强与规模化

- 多模型分析、来源质量评分、反证检索。
- 语义搜索、相似事件和趋势预警。
- 报告自动重构建议和历史版本回溯。
- 对接企业微信、邮件和内部知识库。

## 12. 验收指标

- 公众号链接进入后，90% 可在 3 分钟内生成可审核候选；解析失败必须有明确回退路径。
- 100% 已发布结论可下钻到原始证据和链接。
- 领导首页 3 分钟内可完成今日重点阅读与批示。
- 新证据到报告候选变更的中位时间小于 24 小时。
- 高重要度观点至少包含两个独立来源，或明确标记单一来源风险。
- 产品升级建议均能映射竞对版本证据、科杰现状和责任人。
- 动态报告每次发布均可查看差异、理由、证据和审核记录。

## 13. 本轮三种视觉探索

1. **领导晨报台**：以今日三条信号、对科杰启发、待决事项和报告增量为首屏，最适合领导高频使用。
2. **证据研究台**：以信源、证据链、主题综合和观点升级为核心，最适合研究编辑深度工作。
3. **共创报告室**：以动态报告为中心，左侧证据/观点、中央正文差异、右侧协作审核，最适合跨部门共同维护结论。

建议最终产品不是三选一地完全舍弃另外两种，而是以“领导晨报台”作为首页、以“证据研究台”作为研究工作区、以“共创报告室”作为报告中心。本轮仍需先选择一个视觉主方向，再统一三类工作区的视觉系统。
